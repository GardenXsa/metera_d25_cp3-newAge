Вот обновлённая шпаргалка, дополненная шагами для **связывания локальной папки с новым удалённым репозиторием** и первой отправки файлов.

---

### 🚀 Git-шпаргалка (расширенная)

#### 1️⃣ Начало работы: связать пустой репозиторий на GitHub с папкой на ПК

```bash
# Перейти в нужную папку с проектом
cd путь/к/папке

# Превратить папку в Git-репозиторий
git init

# Добавить все файлы в отслеживание
git add .

# Сделать первый коммит
git commit -m "Первый коммит: все файлы проекта"

# Добавить удалённый репозиторий (ссылка с GitHub)
git remote add origin https://github.com/ваш_логин/имя_репозитория.git

# (опционально) Переименовать основную ветку в main, если нужно
git branch -M main

# Отправить всё на GitHub (в первый раз с ключом -u, чтобы запомнить связь)
git push -u origin main
```

#### 2️⃣ Ежедневная работа

**Просмотр состояния**  
```bash
git status
```

**Добавить все новые/изменённые файлы**  
```bash
git add .
```

**Сохранить изменения локально (коммит)**  
```bash
git commit -m "описание того, что сделано"
```

**Отправить коммиты на GitHub**  
```bash
git push
```

**Забрать последнюю версию с GitHub (если работаете с кем-то или с другого ПК)**  
```bash
git pull
```
> ⚠️ Перед `pull` обязательно закоммитьте свои локальные изменения, иначе возможны конфликты.

**Посмотреть историю коммитов**  
```bash
git log
```

---

### 🔁 Типичный сценарий «нулевой репозиторий → всё на GitHub»

1. Создаёте **пустой репозиторий** на GitHub (без README, .gitignore, лицензии).
2. На ПК в папке с проектом выполняете:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin <ссылка_на_GitHub>
   git push -u origin main
   ```
3. Готово! Все файлы загружены. Далее работаете через `add`, `commit`, `push`, `pull`.
